Section 1

Platform Overview

How Tern is structured and what lives where.

Where you work

Trips

Each client booking lives here. Tasks, automations, emails, documents, and the client-facing itinerary are all tied to the trip. The trip status controls which automations fire automatically.

Your toolbox

Library

Everything reusable lives here: Workflows, Email Templates, Photos, and Suppliers. Changes made in Library update all future uses — but not existing trips already in progress.

Team communication

Email Tab (per Trip)

Connect your Gmail / Outlook to send and track all client emails from inside Tern. Use Apply Template to pull from your library. Use Shared With Me for TBB cruise-line specific emails.

⭐

Key rule Always Publish the trip after making pricing or content changes. A small red dot on the Publish button means the client is seeing an older version. Publishing commits all updates to the client-facing view.

Section 5

Managing Emails in Tern

Where emails live, how to edit them, and what the difference is between template types.

Your editable templates

Non-TBB Emails

Library → Emails → Non-TBB Emails. These are your custom templates (Arrival Check-In, Bon Voyage, Welcome Home, etc.). Edit here and click "Apply Updates" to push changes to all workflows using that template.

Cruise-line specific

Shared With Me (TBB)

Library → Emails → Shared With Me. Search the cruise line name (e.g. "royal"). These are maintained by Travel Biz Boss and update automatically — don't save your own copy or you'll lose the auto-updates. Includes deposit & final payment emails per cruise line.

Cruise workflow emails

Edit Directly in Workflow

The 180-day, 90-day, 60-day, etc. cruise emails are not in the email template library — they live inside the workflow itself. Edit them via Library → Workflows → Cruise Mass Market Booked → click the automation.

Sending ad-hoc emails

Per-Trip Email Tab

Open a trip → click Email in the left nav → click New → click Apply Template. For cruise deposit/final payment: click Shared With Me and search the cruise line.

📌

Remember Editing an email in the Library updates the template globally. Editing an email inside a specific trip's automation only changes it for that trip. Use trip-level edits for personalization; use Library edits for permanent changes you want everywhere.

Section 6

Client Forms

Two forms for two scenarios. Send manually from the trip — do not automate for existing clients.

For international / passport trips

Long Form

Use for FIT, international cruises, and any trip requiring passport details. Captures everything needed for complex trip planning.

Includes: Name, DOB, email, home address, loyalty numbers, passport info (valid dates, pages), food allergies, special requests, travel history, preferences.

For quick / domestic / mass-market cruises

Short / Quick Form

Use for straightforward mass-market cruises, hotel-only bookings, and clients you know well.

Includes ONLY: Names (all guests in cabin), dates of birth, email addresses, home addresses, loyalty/rewards numbers.

⚠️

Do NOT auto-send the Long Form to existing clients. The Long Form automation is turned off in the Inbound New workflow and is not included in Inbound Existing. Send it manually via the task prompt, and choose which form is appropriate. Sending a profile form to a 10-year client who already has everything on file feels insulting.

Section 8

Glossary

Key terms used in Tern and in these cheat sheets.

TermWhat It Means
WorkflowA bundled set of tasks + automations that applies to a trip. Multiple workflows can stack on one trip.
Auto-ApplyWhen enabled on a workflow, it fires automatically the moment a trip reaches the right status or gets the right tag. No manual "apply" needed.
AutomationAn individual scheduled email or action within a workflow. Can be Active (fires automatically) or Inactive (you decide when/if to activate).
TaskA to-do item assigned to you (or a VA). Tasks go to the team, not clients. Complete, reassign, or delete them as needed.
TBBTravel Biz Boss — the third-party workflow system Tern is built on top of. TBB emails live in "Shared With Me" and update automatically.
PublishCommits your trip changes to the client-facing view. A red dot on the Publish button = unpublished changes the client cannot see yet.
Trip StatusInbound New → Inbound Existing → Planning → Booked → Prep for Travel → Traveling → Traveled. Status changes trigger auto-apply workflows.
Trip TagLabels added to a trip (e.g. "Mass Market Cruise", "FIT") that trigger tag-specific workflows.
Shared With MeTBB's email library section. Contains cruise-line specific deposit, final payment, and other branded emails. Do not save your own copy.
Apply UpdatesButton in the email template editor. Pushes your changes to all workflows currently using that template. Must click this after editing a template.
2FATwo-Factor Authentication. Must be enabled in Tern settings before the "Payment Due" trigger becomes available (needed for the 7-day payment reminder).
OBCOnboard Credit — a gift/perk added to the client's cruise booking. Task fires 14 days before trip.
ETIAElectronic Travel Information Authorization — like ESTA for EU/UK destinations. Required for some international cruise ports.
FITFully Independent Travel — custom-built land itineraries (vs. packaged cruises). More planning-intensive; uses the FIT Booked workflow.
SionLisa's host agency commission tracking system. Enter bookings in both Tern and Sion. Tern commission fields are optional but useful for cross-referencing.
